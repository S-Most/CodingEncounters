for...of vs. for...in: What's the Difference?

November 15, 2025

In JavaScript, both for...of and for...in are loops for iteration, but they serve fundamentally different purposes. Understanding their distinction is key to avoiding common bugs. The core difference is this:

  • for...of iterates over the values of iterable objects (like Arrays, Strings, Maps, etc.).
  • for...in iterates over the enumerable property keys of an object.

The for...of Loop: For Values of Iterables

Introduced in ES6, the for...of loop is the modern and preferred way to iterate over iterable collections. It provides a simple, clean syntax to get the value of each item directly. Use it for arrays, strings, maps, sets, and other iterable data structures. 

const languanges = ['JavaScript', 'C#', 'TypeScript'];

for (const fruit of fruits) {
  console.log(fruit);
}
// Output:
// "JavaScript"
// "C#"
// "TypeScript"

The for...in Loop: For Keys of Objects

The for...in loop is designed to iterate over the property names (keys) of a plain object. This makes it useful for inspecting the properties of an object. It's important to know that this loop will also iterate over inherited enumerable properties from the object's prototype chain. 

const person = {
  name: 'Nexy',
  age: 30,
  job: 'Mascot'
};

for (const key in person) {
  console.log(`${key}: ${person[key]}`);
}
// Output:
// "name: Nexy"
// "age: 30"
// "job: Mascot"

The Pitfall: Why You Shouldn't Use for...in on Arrays

While you can technically use a for...in loop on an array, it is strongly discouraged. The reason is that it iterates over the index keys as strings ('0', '1', '2'), not the actual values. Worse, it will also pick up any non-element properties, including those on the `Array.prototype`, which can lead to unexpected behavior.

Warning: Avoid using for...in to iterate over arrays. Use for...of or array methods like forEach() instead. 
const numbers = [10, 20, 30];
Array.prototype.customProperty = () => "Do not do this!";

for (const index in numbers) {
  console.log(index, typeof index);
}
// Output:
// "0" "string"
// "1" "string"
// "2" "string"
// "customProperty" "string"  <-- Unwanted inherited property!

delete Array.prototype.customProperty; // Cleanup
