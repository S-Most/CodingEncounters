CSS Grid Template Areas: Building Complex Layouts with Named Regions

November 19, 2025

Grid template areas are one of CSS Grid's most powerful features. They let you create complex layouts using named regions, making your CSS more readable and maintainable. Let's explore how to use them effectively.

What are Grid Template Areas?

Grid template areas allow you to name sections of your grid and assign elements to those named areas. This creates a visual representation of your layout right in your CSS.

.container {
  display: grid;
  grid-template-areas:
    "header header header"
    "sidebar main main"
    "footer footer footer";
  grid-template-columns: 200px 1fr 1fr;
  grid-template-rows: auto 1fr auto;
  gap: 1rem;
  min-height: 100vh;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }
┌─────────────────────────────────┐
│ header │
├─────────┬───────────────────────┤
│ sidebar │ main │
│ │ │
├─────────┴───────────────────────┤
│ footer │
└─────────────────────────────────┘

Basic Syntax

/* Define the grid template */
.container {
  display: grid;
  grid-template-areas:
    "area1 area2"
    "area3 area3";
}

/* Assign elements to areas */
.item1 { grid-area: area1; }
.item2 { grid-area: area2; }
.item3 { grid-area: area3; }
Key Rules:
  • Each string represents a row
  • Each word in a string represents a column
  • Same name = same area (spans multiple cells)
  • Use a dot (.) for empty cells

Common Layout Patterns

1. Classic Website Layout

.page {
  display: grid;
  grid-template-areas:
    "header header header"
    "nav content sidebar"
    "footer footer footer";
  grid-template-columns: 200px 1fr 300px;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
  gap: 1rem;
}

header  { grid-area: header; }
nav     { grid-area: nav; }
main    { grid-area: content; }
aside   { grid-area: sidebar; }
footer  { grid-area: footer; }

    "sidebar header header header"

.chart2  { grid-area: chart2; }
.chart3  { grid-area: chart3; }

Responsive Layouts

Grid template areas shine in responsive design. You can completely reorganize your layout at different breakpoints:

/* Mobile: Single column */
.container {
  display: grid;
  grid-template-areas:
    "header"
    "nav"
    "content"
    "sidebar"
    "footer";
  grid-template-columns: 1fr;
}

/* Tablet: Two columns */
@media (min-width: 768px) {
  .container {
    grid-template-areas:
      "header header"
      "nav nav"
      "content sidebar"
      "footer footer";
    grid-template-columns: 2fr 1fr;
  }
}

/* Desktop: Three columns */
@media (min-width: 1024px) {
  .container {
    grid-template-areas:
      "header header header"
      "nav content sidebar"
      "footer footer footer";
    grid-template-columns: 200px 1fr 300px;
  }
}
Pro Tip: Notice how the HTML order doesn't change—only the grid template areas! This is great for accessibility and SEO.

Advanced Techniques

Overlapping Areas

You can create overlapping areas by using grid-area with line numbers:

.container {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-template-rows: repeat(4, 100px);
}

.background {
  grid-area: 1 / 1 / 5 / 5; /* Covers entire grid */
  z-index: 0;
}

.content {
  grid-area: 2 / 2 / 4 / 4; /* Centered, smaller */
  z-index: 1;
}

Combining with Other Grid Properties

.container {
  display: grid;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
  grid-template-columns: minmax(200px, 1fr) 3fr;
  grid-template-rows: auto 1fr auto;
  gap: clamp(1rem, 2vw, 2rem);
}

/* Items can still use grid-column/grid-row */
.special-item {
  grid-area: main;
  justify-self: center;
  align-self: start;
}

Real-World Example: Blog Layout

.blog-post {
  display: grid;
  grid-template-areas:
    "title title title"
    "meta meta meta"
    "featured featured sidebar"
    "content content sidebar"
    "comments comments sidebar"
    "footer footer footer";
  grid-template-columns: 1fr 1fr 300px;
  grid-template-rows: auto auto auto 1fr auto auto;
  gap: 2rem;
  max-width: 1200px;
  margin: 0 auto;
  padding: 2rem;
}

.post-title    { grid-area: title; }
.post-meta     { grid-area: meta; }
.featured-img  { grid-area: featured; }
.post-content  { grid-area: content; }
.comments      { grid-area: comments; }
.sidebar       { grid-area: sidebar; }
.post-footer   { grid-area: footer; }

/* Responsive */
@media (max-width: 768px) {
  .blog-post {
    grid-template-areas:
      "title"
      "meta"
      "featured"
      "content"
      "sidebar"
      "comments"
      "footer";
    grid-template-columns: 1fr;
  }
}

Common Mistakes to Avoid

  • Non-rectangular areas: All cells with the same name must form a rectangle
  • Inconsistent columns: Each row must have the same number of columns
  • Typos in names: Area names are case-sensitive
  • Forgetting quotes: Each row must be in quotes
/* ❌ Wrong - non-rectangular */
grid-template-areas:
  "header header"
  "main main"
  "main sidebar"; /* 'main' is not rectangular! */

/* ✅ Correct */
grid-template-areas:
  "header header"
  "main sidebar"
  "main sidebar";

Browser Support and Fallbacks

Grid template areas have excellent browser support (all modern browsers). For older browsers, provide a fallback:

/* Fallback for old browsers */
.container {
  display: flex;
  flex-direction: column;
}

/* Grid for modern browsers */
@supports (display: grid) {
  .container {
    display: grid;
    grid-template-areas:
      "header"
      "main"
      "footer";
  }
}

Best Practices

  • Use descriptive, semantic names for areas
  • Keep the visual representation readable (align columns)
  • Use template areas for major layout sections
  • Combine with other grid properties for fine control
  • Test responsive layouts at all breakpoints
  • Consider accessibility—HTML order matters for screen readers

Conclusion

Grid template areas make complex layouts intuitive and maintainable. By naming your grid regions, you create self-documenting CSS that's easy to understand and modify. They're especially powerful for responsive design, allowing you to completely reorganize layouts without changing HTML structure.

Start using grid template areas in your next project—you'll wonder how you ever lived without them!
